================================================================
檔案：backend/services/alert_service.py
功能：儀器警報服務。查詢 radarFileCheck 快照資料表，計算每台儀器的
      資料延遲（diff_time_minutes），並與閾值比較來判斷是否觸發警報。
      同時管理記憶體中的每台儀器閾值設定。
================================================================

           並與閾值比較。

--- 模組級變數 (第 19-33 行) ---

第 22-23 行：
    `# Runtime threshold store: file_type -> threshold (minutes)`
    `# Initialised from config.yaml on first access; persists in memory until restart.`
    說明：_thresholds 是記憶體中的閾值字典，程式啟動後從 config.yaml 初始化，
    執行期間可透過 API 動態修改，但重啟後會重設。

第 23 行：`_thresholds: dict[str, float] = {}`
          模組級私有字典：鍵為 FileType（如 "RADAR_A"），值為閾值（分鐘）。
          空字典表示尚未從 config 初始化。

第 25-33 行：`_INSTRUMENT_STATUS_SQL = text("""...""")`
             SQL 查詢字串常數（提前定義，避免每次呼叫重複解析）：
               SELECT ftl.FileType          → 儀器識別碼
               SELECT ftl.EquipmentName     → 儀器名稱
               SELECT rfc.FileTime          → 最新檔案的 Unix 時間戳記
               TIMESTAMPDIFF(SECOND, FROM_UNIXTIME(rfc.FileTime), NOW()) / 60.0
                                            → 計算距今的時間差（分鐘）
               FROM FileTypeList ftl        → 儀器主列表
               LEFT JOIN radarFileCheck rfc → 左連接最新快照
                 ON ftl.FileType = rfc.FileType
               LEFT JOIN 確保即使 radarFileCheck 沒有對應記錄，
               FileTypeList 中的儀器仍會出現在結果中（FileTime 為 NULL）。

第 75-85 行：處理「無資料」情況：
             若 FileTime 或 diff_time_minutes 為 NULL（LEFT JOIN 無對應記錄），
             建立 is_alert=True 的 InstrumentStatus（缺資料視為警報），
             latest_file_time 和 diff_time_minutes 均為 None，
             然後 continue 跳過後續正常處理。

第 87 行：`diff = max(0.0, float(row.diff_time_minutes))`
          將 diff_time_minutes 轉為浮點數，並確保不小於 0。

第 88 行：`latest_file_time = datetime.fromtimestamp(float(row.FileTime), tz=timezone.utc)`
          將 Unix 時間戳（秒數）轉換為有時區資訊的 UTC datetime 物件。
          float() 轉換確保資料型別正確。

第 90-97 行：建立正常情況的 InstrumentStatus：
             is_alert=diff > threshold → 若延遲超過閾值則觸發警報。

第 99 行：`logger.info("get_all_instrument_statuses: %d instruments queried", len(statuses))`
          記錄查詢到的儀器數量。
