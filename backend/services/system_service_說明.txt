================================================================
檔案：backend/services/system_service.py
功能：系統狀態查詢服務。
      1. get_system_status()：從 SystemStatus DB 查詢各電腦的 CPU 負載與記憶體使用率。
      2. get_disk_status()：從 DiskStatus DB 查詢各電腦的磁碟使用量。
================================================================

--- 模組說明字串 (第 1-3 行) ---
第 1-3 行：docstring，說明此服務查詢 SystemStatus 和 DiskStatus 資料庫。

--- 匯入區塊 (第 4-9 行) ---
第 4 行：`from __future__ import annotations`
         啟用延遲型別評估。

第 5 行：`import logging`
         日誌模組。

第 6 行：`from sqlalchemy import text`
         SQL 字串包裝器，確保 SQL 以安全方式執行。

第 7 行：`from sqlalchemy.exc import OperationalError, SQLAlchemyError`
         捕捉資料庫操作錯誤和 SQLAlchemy 框架錯誤。

第 8 行：`from backend.config import get_config`
         取得應用程式設定（查詢逾時時間）。

第 9 行：`from backend.database import get_session`
         取得資料庫 Session 的 context manager。

--- 模組級變數 (第 11-22 行) ---
第 11 行：`logger = logging.getLogger("system_service")`
          建立此服務的日誌紀錄器。

第 13-17 行：`_SYSTEM_SQL = text("""...""")`
             系統狀態查詢 SQL 常數：
               cl.IP            → 電腦 IP 位址（主鍵識別）
               cl.Load_1        → 最近 1 分鐘平均 CPU 負載（Linux uptime 值）
               cl.Load_5        → 最近 5 分鐘平均 CPU 負載
               cl.LOAD_15       → 最近 15 分鐘平均 CPU 負載（注意：欄位名稱大小寫與 Load_1/Load_5 不同）
               cl.MemoryUSE     → 記憶體使用率（百分比，0~100）
               sl.EquipmentName → 設備名稱（從 SystemIPList 取得）
               FROM CheckList cl        → 目前最新狀態快照資料表
               LEFT JOIN SystemIPList sl ON cl.IP = sl.IP
                                        → 左連接設備名稱對應表
                                          若 IP 不在 SystemIPList 中，EquipmentName 為 NULL

第 19-22 行：`_DISK_SQL = text("""...""")`
             磁碟狀態查詢 SQL 常數：
               IP         → 電腦 IP 位址
               FileSystem → 磁碟分割區或檔案系統路徑（如 /dev/sda1）
               Used       → 磁碟使用量（通常為百分比或 GB）
               FROM CheckList → DiskStatus 資料庫的快照資料表（注意：兩個 DB 都有名為 CheckList 的表）

--- get_system_status 函式 (第 25-43 行) ---
第 25 行：`def get_system_status() -> list[dict]:`
          公開函式，回傳所有電腦的系統狀態，每台電腦一個字典。

第 26 行：`timeout = get_config().system.query_timeout_seconds`
          從設定取出查詢逾時（秒），避免 DB 長時間無回應。

第 27-29 行：`with get_session("system_status") as session:`
             開啟 system_status 資料庫的 Session，執行查詢並取得結果。
             .fetchall() 一次取出所有記錄。

第 30-40 行：列表推導式，將每筆查詢結果轉為字典：
    "ip"           → row.IP（電腦的 IP 位址）
    "equipment_name" → row.EquipmentName or row.IP
                     若 SystemIPList 沒有對應的設備名稱（EquipmentName 為 NULL），
                     改用 IP 位址作為顯示名稱（確保前端不顯示空白）。
    "load_1"       → float(row.Load_1) if row.Load_1 is not None else None
                     有資料時轉為浮點數，NULL 時保留 None（前端需處理 null 顯示）。
    "load_5"       → 同上，5 分鐘負載。
    "load_15"      → 同上，15 分鐘負載（注意欄位名稱 LOAD_15 為全大寫）。
    "memory_use"   → 記憶體使用率，同樣處理 NULL 的情況。

第 41-43 行：`except (OperationalError, SQLAlchemyError) as exc:`
             DB 連線失敗時，記錄 ERROR 日誌，回傳空列表（服務降級）。
             前端收到空列表 [] 應顯示「無法取得資料」而非程式崩潰。

--- get_disk_status 函式 (第 46-61 行) ---
第 46 行：`def get_disk_status() -> list[dict]:`
          公開函式，回傳所有電腦的磁碟狀態，每筆記錄一個字典。
          注意：一台電腦可能有多個磁碟分割，所以結果筆數可能多於電腦台數。

第 47 行：`timeout = get_config().system.query_timeout_seconds`
          取出查詢逾時設定。

第 48-50 行：`with get_session("disk_status") as session:`
             開啟 disk_status 資料庫（與 system_status 不同的 DB）的 Session，執行查詢。

第 51-57 行：列表推導式，每筆結果轉為字典：
    "ip"         → row.IP（電腦 IP）
    "file_system" → row.FileSystem（磁碟分割路徑）
    "used"       → float(row.Used) if row.Used is not None else None
                  磁碟使用量，NULL 時保留 None。

第 58-60 行：`except (OperationalError, SQLAlchemyError) as exc:`
             DB 錯誤時記錄日誌並回傳空列表（服務降級，與 get_system_status 一致的錯誤處理模式）。
