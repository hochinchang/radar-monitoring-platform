================================================================
檔案：backend/database.py
功能：管理三個 MySQL 資料庫（file_status、system_status、disk_status）的
      SQLAlchemy 連線池，提供統一的 Session 存取介面與連線健康檢查。
================================================================

--- 模組說明字串 (第 1-4 行) ---
第 1-4 行：docstring，說明此模組管理三個獨立的連線池。

--- 匯入區塊 (第 6-18 行) ---
第 6 行：`from __future__ import annotations`
         啟用延遲型別評估，支援新式型別語法。

第 8 行：`import logging`
         匯入標準日誌模組，用於記錄連線狀態與錯誤。

第 9 行：`import time`
         匯入時間模組，用於連線重試之間的等待（time.sleep）。

第 10 行：`from contextlib import contextmanager`
          匯入 contextmanager 裝飾器，將函式轉為 with 語法可用的 context manager。

第 11 行：`from typing import Dict, Generator`
          匯入 Dict 字典泛型與 Generator 產生器泛型，用於型別標注。

第 13 行：`from sqlalchemy import create_engine, text`
          create_engine → 建立 SQLAlchemy Engine（含連線池）
          text          → 將原始 SQL 字串包裝成安全的 SQLAlchemy 文字物件。

第 14 行：`from sqlalchemy.engine import Engine`
          Engine 類別型別，用於型別標注。

第 15 行：`from sqlalchemy.orm import Session, sessionmaker`
          Session     → 資料庫工作單元，負責執行查詢與管理交易。
          sessionmaker → Session 工廠，可批量建立 Session 實例。

第 16 行：`from sqlalchemy.exc import OperationalError, SQLAlchemyError`
          OperationalError → 資料庫連線失敗、逾時等操作層錯誤。
          SQLAlchemyError  → 所有 SQLAlchemy 錯誤的基底類別。

第 18 行：`from backend.config import get_config`
          匯入設定存取函式，取得資料庫連線參數。

--- 模組初始化 (第 20-26 行) ---
第 20 行：`logger = logging.getLogger("database")`
          建立名為 "database" 的日誌紀錄器，日誌輸出會帶此前綴。

第 22 行：`_VALID_DB_NAMES = ("file_status", "system_status", "disk_status")`
          定義合法資料庫名稱的元組常數，用於輸入驗證。
          前綴底線表示模組私有。

第 25 行：`_engines: Dict[str, Engine] = {}`
          模組級字典，存放各資料庫名稱對應的 Engine 實例。
          採「延遲初始化」（Lazy Initialization）：首次存取時才建立，而非程式啟動時。

第 26 行：`_session_factories: Dict[str, sessionmaker] = {}`
          模組級字典，存放各資料庫名稱對應的 sessionmaker 工廠。
          與 _engines 一一對應。

--- _build_url 函式 (第 29-34 行) ---
第 29 行：`def _build_url(db_name: str) -> str:`
          私有函式，根據設定組裝 SQLAlchemy 資料庫連線 URL。

第 30 行：`cfg = get_config().databases[db_name]`
          從 AppConfig 取出指定資料庫的 DatabaseConfig。

第 31-34 行：`return f"mysql+pymysql://..."`
             組裝標準的 SQLAlchemy 連線 URL，格式為：
             mysql+pymysql://帳號:密碼@主機:埠/資料庫名稱
             mysql+pymysql 指定使用 PyMySQL 驅動程式連接 MySQL。

--- _init_engine 函式 (第 37-49 行) ---
第 37 行：`def _init_engine(db_name: str) -> Engine:`
          私有函式，為指定資料庫建立 Engine 並註冊到模組級字典中。

第 39 行：`cfg = get_config().databases[db_name]`
          取出資料庫設定（含連線池大小）。

第 40 行：`url = _build_url(db_name)`
          呼叫 _build_url 取得連線 URL 字串。

第 41-45 行：`engine = create_engine(url, pool_size=..., pool_pre_ping=True)`
             建立 SQLAlchemy Engine：
               pool_size    → 連線池最大連線數，從設定檔讀取。
               pool_pre_ping=True → 每次取用連線前先發送 "SELECT 1" 測試，
                                    確保連線有效，避免使用到已斷開的舊連線。

第 46 行：`_engines[db_name] = engine`
          將新建的 Engine 存入模組級字典，供後續呼叫直接取用。

第 47 行：`_session_factories[db_name] = sessionmaker(bind=engine)`
          建立綁定此 Engine 的 sessionmaker 工廠，存入字典。

第 48 行：`logger.info("Engine initialised for database '%s'", db_name)`
          記錄 Engine 初始化成功的 INFO 日誌。

第 49 行：`return engine`
          回傳剛建立的 Engine。

--- _validate_db_name 函式 (第 52-56 行) ---
第 52 行：`def _validate_db_name(db_name: str) -> None:`
          私有函式，驗證傳入的資料庫名稱是否合法。

第 53-56 行：若 db_name 不在 _VALID_DB_NAMES 中，拋出 ValueError 並附上清楚的錯誤訊息，
             防止因拼字錯誤存取不存在的資料庫。

--- get_engine 函式 (第 59-64 行) ---
第 59 行：`def get_engine(db_name: str) -> Engine:`
          公開函式，取得指定資料庫的 Engine，首次呼叫時延遲初始化。

第 61 行：`_validate_db_name(db_name)`
          先驗證名稱合法性。

第 62-63 行：`if db_name not in _engines: _init_engine(db_name)`
             若 Engine 尚未建立，呼叫 _init_engine 建立並快取。

第 64 行：`return _engines[db_name]`
          回傳已快取的 Engine 實例。

--- get_session context manager (第 67-82 行) ---
第 67 行：`@contextmanager`
          裝飾器，將下方的產生器函式轉為 context manager，
          支援 with get_session("...") as session 語法。

第 68 行：`def get_session(db_name: str) -> Generator[Session, None, None]:`
          公開函式，提供安全的資料庫 Session，自動管理 commit/rollback/close。
          回傳型別標注：Generator[產出型別, 傳入型別, 回傳型別]。

第 70 行：`_validate_db_name(db_name)`
          驗證名稱合法性。

第 71-72 行：若 sessionmaker 尚未建立（例如 _init_engine 未被呼叫過），
             先呼叫 _init_engine 確保工廠存在。

第 73 行：`factory = _session_factories[db_name]`
          取出 sessionmaker 工廠。

第 74 行：`session: Session = factory()`
          呼叫工廠建立一個新的 Session 實例。

第 75 行：`try:`
          開始交易保護區塊。

第 76 行：`yield session`
          將 Session 交給 with 區塊使用，程式執行流程暫停在此處。

第 77 行：`session.commit()`
          with 區塊正常結束後，提交交易（確保寫入操作生效）。
          本系統為唯讀查詢，commit 不會有副作用，但保持一致性。

第 78-80 行：`except Exception: session.rollback(); raise`
             若 with 區塊內發生任何例外：
               - 回滾交易（撤銷未完成的寫入）
               - 重新拋出原始例外，讓上層處理

第 81-82 行：`finally: session.close()`
             無論成功或失敗，最終都關閉 Session，將連線歸還連線池。

--- check_connection 函式 (第 85-125 行) ---
第 85 行：`def check_connection(db_name: str) -> bool:`
          公開函式，嘗試連線至指定資料庫，支援重試邏輯。
          回傳 True 表示連線成功，False 表示所有重試均失敗。

第 93 行：`_validate_db_name(db_name)`
          驗證名稱合法性。

第 94 行：`sys_cfg = get_config().system`
          取出系統設定（含重試次數與間隔）。

第 95 行：`max_attempts: int = sys_cfg.max_reconnect_attempts`
          從設定取出最大重試次數。

第 96 行：`interval: int = sys_cfg.reconnect_interval_seconds`
          從設定取出重試間隔秒數。

第 98 行：`engine = get_engine(db_name)`
          取得（或建立）Engine。

第 100 行：`for attempt in range(1, max_attempts + 1):`
           從 1 開始計數，遍歷至最大重試次數（含），提供人類可讀的嘗試編號。

第 101-103 行：`with engine.connect() as conn: conn.execute(text("SELECT 1"))`
              嘗試開啟連線並執行最簡單的查詢 "SELECT 1"，
              若成功表示資料庫可連通。

第 104-108 行：連線成功，記錄 INFO 日誌，立即回傳 True。

第 109-113 行：`except (OperationalError, SQLAlchemyError) as exc:`
              捕捉連線相關錯誤，記錄 WARNING 日誌（包含嘗試次數與錯誤訊息）。

第 114-119 行：`if attempt < max_attempts: time.sleep(interval)`
              若尚有剩餘重試次數，記錄 INFO 日誌後等待指定秒數再重試。
              最後一次失敗時不等待（沒有意義）。

第 121-124 行：所有重試均失敗後，記錄 ERROR 日誌，回傳 False。
