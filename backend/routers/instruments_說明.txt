================================================================
檔案：backend/routers/instruments.py
功能：定義 /api/v1/instruments/* 的 REST API 端點。
      負責列出所有儀器，以及更新個別儀器的 Max_DiffTime_Threshold（最大時間差閾值）。
================================================================

--- 模組說明字串 (第 1-4 行) ---
第 1-4 行：docstring，說明路由器管理每個儀器的 Max_DiffTime_Threshold 設定。

--- 匯入區塊 (第 6-23 行) ---
第 6 行：`from __future__ import annotations`
         啟用延遲型別評估。

第 8 行：`import logging`
         匯入日誌模組。

第 9 行：`from datetime import datetime, timezone`
         datetime → 用於回應中的 updated_at 時間戳記。
         timezone → 使用 timezone.utc 確保時間為 UTC。

第 11 行：`from fastapi import APIRouter, HTTPException`
          APIRouter    → 路由群組物件。
          HTTPException → 用於拋出 404（儀器不存在）等 HTTP 錯誤。

第 13-18 行：匯入四個 Pydantic 模型：
             InstrumentListItem        → 列表中單一儀器的資料結構
             InstrumentListResponse    → GET /instruments 的整體回應包裝
             InstrumentThresholdSetting → PUT 請求的 Body 模型（含閾值驗證）
             ThresholdUpdateResponse   → PUT 成功後的回應結構

第 19-23 行：匯入三個 alert_service 函式：
             get_instrument_threshold  → 取得單一儀器的目前閾值
             list_instruments          → 取得所有儀器及其閾值
             set_instrument_threshold  → 更新單一儀器的閾值

--- 路由器初始化 (第 25-26 行) ---
第 25 行：`logger = logging.getLogger("routers.instruments")`
          建立此路由模組的日誌紀錄器。

第 26 行：`router = APIRouter(prefix="/api/v1/instruments", tags=["instruments"])`
          建立路由器，所有路徑都加上 /api/v1/instruments 前綴。

--- GET /api/v1/instruments 端點 (第 29-42 行) ---
第 29 行：`@router.get("", response_model=InstrumentListResponse)`
          綁定 GET /api/v1/instruments（空路徑 + 前綴 = 完整路徑）。
          response_model 確保回應符合 InstrumentListResponse 結構。

第 30 行：`def get_instruments() -> InstrumentListResponse:`
          無參數端點，直接回傳所有儀器清單。

第 31 行：docstring 說明：回傳所有儀器及其目前的 Max_DiffTime_Threshold。

第 32 行：`items = list_instruments()`
          呼叫 alert_service.list_instruments()，查詢 FileTypeList 資料表，
          取得儀器清單（每筆含 file_type、equipment_name、max_diff_time_threshold）。

第 33-42 行：`return InstrumentListResponse(instruments=[...])`
             用列表推導式，將 list_instruments() 回傳的原始字典列表
             轉換為 InstrumentListItem Pydantic 物件，包裝後回傳。
             逐欄對應：
               i["file_type"]                → InstrumentListItem.file_type
               i["equipment_name"]           → InstrumentListItem.equipment_name
               i["max_diff_time_threshold"]  → InstrumentListItem.max_diff_time_threshold

--- PUT /api/v1/instruments/{file_type}/threshold 端點 (第 45-68 行) ---
第 45 行：`@router.put("/{file_type}/threshold", response_model=ThresholdUpdateResponse)`
          綁定 PUT /api/v1/instruments/{file_type}/threshold。
          {file_type} 是路徑參數，如 /api/v1/instruments/RADAR_A/threshold。

第 46-49 行：`def update_threshold(file_type: str, body: InstrumentThresholdSetting)`
             端點函式，接收兩個來源的參數：
               file_type → 路徑參數（URL 中的 {file_type}）
               body      → 請求 Body（JSON），Pydantic 自動解析並驗證
                           若 body.max_diff_time_threshold < 0，Pydantic 自動回 422

第 50-53 行：docstring 說明：
             - 更新指定儀器的 Max_DiffTime_Threshold
             - file_type 不存在時回傳 404
             - threshold 為負數時由 Pydantic Field(ge=0) 回傳 422

第 56 行：`instruments = list_instruments()`
          重新查詢儀器清單，取得目前所有已知的儀器。

第 57 行：`known = {i["file_type"] for i in instruments}`
          用集合推導式建立 file_type 的集合（Set），
          集合的 in 運算是 O(1) 查找，比列表更高效。

第 58-59 行：`if file_type not in known: raise HTTPException(status_code=404, ...)`
             若請求的 file_type 不在已知儀器列表中，
             拋出 404 Not Found，並附上中文錯誤訊息。

第 61 行：`set_instrument_threshold(file_type, body.max_diff_time_threshold)`
          呼叫 alert_service.set_instrument_threshold() 更新記憶體中的閾值。
          注意：此更新僅存於記憶體，服務重啟後會重設為 config.yaml 中的值。

第 62 行：`logger.info("Threshold updated via API: %s -> %.1f", file_type, body.max_diff_time_threshold)`
          記錄閾值更新操作的 INFO 日誌，提供操作審計紀錄。
          %.1f 格式化浮點數至一位小數。

第 64-68 行：`return ThresholdUpdateResponse(...)`
             組裝並回傳更新確認回應：
               file_type                 → 被更新的儀器識別碼（確認）
               max_diff_time_threshold   → 更新後的新閾值（確認）
               updated_at                → 操作完成的 UTC 時間戳記
